1) 🧠 Problem: One Component Crash = Whole Tree Crash
function BuggyComponent() {
  throw new Error("Something broke");
}

Use:

function App() {
  return (
    <div>
      <h1>Dashboard</h1>
      <BuggyComponent />
    </div>
  );
}

👉 Result:
whole app UI crash ❌

2) 🛡️ Error Boundary Concept

Error boundary catches:

render errors
lifecycle errors
child tree errors

and shows:

fallback UI instead of app crash

3) 🚀 Basic Error Boundary

Important:

Error boundaries are class components

import React from "react";

class ErrorBoundary extends React.Component {
  constructor(props) {
    super(props);
    this.state = { hasError: false };
  }

  static getDerivedStateFromError(error) {
    return { hasError: true };
  }

  componentDidCatch(error, info) {
    console.log("Error:", error);
    console.log("Info:", info);
  }

  render() {
    if (this.state.hasError) {
      return <h2>Something went wrong 😢</h2>;
    }

    return this.props.children;
  }
}
4) 🧩 Usage
<ErrorBoundary>
  <BuggyComponent />
</ErrorBoundary>

Now:
👉 buggy component crash
👉 fallback UI shown ✅
👉 app alive 🚀

5) 🎯 Best Placement Strategy
✅ Page-level
<ErrorBoundary>
  <DashboardPage />
</ErrorBoundary>
✅ Widget-level
<ErrorBoundary>
  <AnalyticsChart />
</ErrorBoundary>

Best for dashboards.

6) ⚠️ What It Does NOT Catch

Error boundary does NOT catch:

event handler errors
async errors
setTimeout
API failures
SSR errors

Example:

<button onClick={() => { throw new Error("fail") }}>

❌ not caught

Need try/catch

7) 🛒 Real App Example
