1) 🧠 Problem: recalculation on every render
function App() {
  const [count, setCount] = useState(0);

  const total = heavyCalculation();

  return (
    <button onClick={() => setCount(count + 1)}>
      {count}
    </button>
  );
}
❌ Problem

Button click করলে:

state update
component re-render
heavyCalculation() আবার run 😵

Even if calculation count-এর সাথে related না.

2) 🚀 useMemo
import { useMemo } from "react";

const total = useMemo(() => {
  return heavyCalculation();
}, []);
✅ Meaning
first render → run
later renders → cached result
dependency change না হলে rerun না
3) 🧠 Real Example
function App() {
  const [count, setCount] = useState(0);
  const [price] = useState(100);

  const total = useMemo(() => {
    console.log("calculating...");
    return price * 1000;
  }, [price]);

  return (
    <button onClick={() => setCount(count + 1)}>
      Count: {count} | Total: {total}
    </button>
  );
}

👉 count change হলে:

component re-render
total recalculated হবে না ✅
4) 🛒 Ecommerce Real Use
const filteredProducts = useMemo(() => {
  return products.filter((p) =>
    p.name.toLowerCase().includes(query.toLowerCase())
  );
}, [products, query]);
✅ Why powerful

large product list হলে every keystroke lag কমায়.

5) 📊 Dashboard Real Use
const chartData = useMemo(() => {
  return orders.reduce((sum, order) => sum + order.total, 0);
}, [orders]);

analytics dashboard এ huge.

6) ⚠️ Common Mistakes
❌ using for everything
const name = useMemo(() => "Nirjhor", []);

useless ❌

❌ missing dependencies
useMemo(() => filter(query), []);

query change হলেও stale result ❌

7) 🧠 When to use

Use only when:

expensive loop
filter/map/reduce large arrays
derived dashboard data
heavy object generation
stable props for memo children
8) 🔥 React.memo + useMemo

super combo:

const users = useMemo(() => generateUsers(), []);
return <UserList users={users} />;

memo child + stable prop reference = max optimization 🚀
