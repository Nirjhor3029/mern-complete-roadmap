1️⃣ 🧠 React Re-render — Reality

React component re-render হয় যখন:

state change
props change
parent re-render
Example
function Parent() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <button onClick={() => setCount(count + 1)}>+</button>
      <Child />
    </div>
  );
}

function Child() {
  console.log("Child rendered");
  return <h2>I am child</h2>;
}

👉 Button click করলে:

Parent re-render
Child also re-render ❗

Even though child has no relation with count.

⚠️ Problem

Unnecessary re-render = performance waste
Large apps এ এটা huge issue.

2️⃣ 🔍 Why Child Re-render?

React by default:

parent re-render → all children re-render

কারণ React doesn't know child needs update or not.

3️⃣ 🚀 React.memo

Solution:

import React from "react";

const Child = React.memo(function Child() {
  console.log("Child rendered");
  return <h2>I am child</h2>;
});

Now:
👉 Parent re-render হলেও child re-render হবে না (if props unchanged)

4️⃣ 🧠 How React.memo Works
shallow compare props
props same → skip render
props changed → re-render
5️⃣ 🧩 With Props Example
const Child = React.memo(function Child({ name }) {
  console.log("Child rendered");
  return <h2>{name}</h2>;
});

Parent:

<Child name="Nirjhor" />

👉 static prop → no re-render

6️⃣ ⚠️ Common Mistake
<Child name={{ value: "Nirjhor" }} />

এটা object → new reference every render ❌
so memo fail করবে.

7️⃣ 🧠 Real Engineer Insight

React.memo use করো when:

✔ heavy component
✔ expensive rendering
✔ stable props

❌ small components → unnecessary complexity

8️⃣ 🛒 Real Example
Product Card
const ProductCard = React.memo(({ product }) => {
  console.log("Rendering:", product.name);
  return <div>{product.name}</div>;
});

এটা ecommerce app এ huge performance gain দেয়.
