1) 🧠 The Problem
function Parent() {
  const [count, setCount] = useState(0);

  function handleClick() {
    console.log("clicked");
  }

  return (
    <>
      <button onClick={() => setCount(count + 1)}>+</button>
      <Child onClick={handleClick} />
    </>
  );
}

Child:

const Child = React.memo(({ onClick }) => {
  console.log("Child rendered");
  return <button onClick={onClick}>Child Btn</button>;
});
❗ Problem

Parent re-render হলে:

function handleClick() {}

new function reference create হয় every render 😵

So memo child thinks:

prop changed

and re-renders ❌

2) 🚀 useCallback
import { useCallback } from "react";

const handleClick = useCallback(() => {
  console.log("clicked");
}, []);

Now:

same function reference reused
child memo works ✅
3) 🧠 Mental Model
useMemo

caches value

useCallback

caches function

Internally:

useCallback(fn, deps)

≈

useMemo(() => fn, deps)
4) 🛒 Real Ecommerce Example
const handleAddToCart = useCallback((id) => {
  setCart((prev) => [...prev, id]);
}, []);

Pass to memoized product cards:

<ProductCard onAdd={handleAddToCart} />

Now 100 product cards unnecessary re-render avoid হবে 🚀

5) ⚠️ Dependency Trap
const logCount = useCallback(() => {
  console.log(count);
}, []);

❌ stale count

Fix:

const logCount = useCallback(() => {
  console.log(count);
}, [count]);
6) 🔥 useCallback + React.memo

Best combo:

const Child = React.memo(({ onDelete }) => {
  return <button onClick={onDelete}>Delete</button>;
});

Parent:

const onDelete = useCallback(() => {
  console.log("deleted");
}, []);

This is real-world optimization pattern.

7) ⚠️ When NOT to use

Don’t use everywhere ❌

Bad:

const sayHi = useCallback(() => console.log("hi"), []);

small components এ unnecessary complexity.

Use only when:

callback props to memo child
expensive child trees
stable deps important
