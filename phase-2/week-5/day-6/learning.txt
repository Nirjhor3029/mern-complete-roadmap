1) 🧠 Problem: duplicated state

ধরো দুইটা input আছে:

temperature in Celsius
temperature in Fahrenheit

যদি দুই child component নিজের নিজের state রাখে,
তাহলে sync problem হবে ❌

এই জন্য:

shared data should move to common parent

এটাই:

lifting state up

2) 🚀 Basic Example
❌ Wrong
function ChildA() {
  const [value, setValue] = useState("");
}
function ChildB() {
  const [value, setValue] = useState("");
}

এরা sync না.

✅ Correct → Parent owns state
import { useState } from "react";

function Parent() {
  const [name, setName] = useState("");

  return (
    <div>
      <InputBox value={name} onChange={setName} />
      <Preview value={name} />
    </div>
  );
}
3) 🧩 Child Receives Props
function InputBox({ value, onChange }) {
  return (
    <input
      value={value}
      onChange={(e) => onChange(e.target.value)}
    />
  );
}

Preview:

function Preview({ value }) {
  return <h2>Hello, {value}</h2>;
}

এখানে:

state → parent
UI → child
update → child → parent → all children sync
4) 🔄 Data Flow Rule

React follows:

one-way data flow

Flow:

Parent state
↓
props
↓
child UI
↓
callback event
↓
parent setter

এইটা খুব important architecture rule.

5) 🎯 Real Life Example
🛒 Quantity Shared State
function Cart() {
  const [qty, setQty] = useState(1);

  return (
    <div>
      <QuantityInput qty={qty} setQty={setQty} />
      <PricePreview qty={qty} />
    </div>
  );
}

Child:

function QuantityInput({ qty, setQty }) {
  return (
    <button onClick={() => setQty(qty + 1)}>
      Increase
    </button>
  );
}

Preview:

function PricePreview({ qty }) {
  return <h3>Total: {qty * 100}</h3>;
}

এটা ecommerce cart-এর real architecture.

6) 🧱 Reusable Component Pattern
function TextInput({ label, value, onChange }) {
  return (
    <div>
      <label>{label}</label>
      <input
        value={value}
        onChange={(e) => onChange(e.target.value)}
      />
    </div>
  );
}

Use:

<TextInput
  label="Name"
  value={name}
  onChange={setName}
/>

এটা real scalable UI pattern 🔥

7) ⚠️ Common Mistakes
❌ state duplicated in siblings

same data multiple child-এ রাখো না.

❌ child mutating props
props.value = "abc";

Never ❌

props are readonly.
