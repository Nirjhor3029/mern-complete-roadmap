1) 🧠 Controlled Component কী?

React-এ best practice হলো:

UI input value = React state

মানে input field-এর value React control করবে.

import { useState } from "react";

function App() {
  const [name, setName] = useState("");

  return (
    <input
      value={name}
      onChange={(e) => setName(e.target.value)}
    />
  );
}

এখানে:

value={name} → input UI reads from state
onChange → state updates from UI

এটাই:

two-way binding style data flow

2) 🔁 Real Two-Way Flow

Flow:

user types → input changes
onChange fires
state updates
component re-renders
new value shows in input

এটা বুঝতে পারলে forms super easy.

4) 🧾 Form Submit
function LoginForm() {
  const [email, setEmail] = useState("");

  function handleSubmit(e) {
    e.preventDefault();
    console.log("Submitted:", email);
  }

  return (
    <form onSubmit={handleSubmit}>
      <input
        value={email}
        onChange={(e) => setEmail(e.target.value)}
      />
      <button type="submit">Login</button>
    </form>
  );
}
✅ Why preventDefault()

না দিলে browser page reload করবে.

5) 🧩 Multiple Inputs (Very Important)
const [form, setForm] = useState({
  name: "",
  email: ""
});

Update:

function handleChange(e) {
  setForm({
    ...form,
    [e.target.name]: e.target.value
  });
}

Use:

<input
  name="name"
  value={form.name}
  onChange={handleChange}
/>

<input
  name="email"
  value={form.email}
  onChange={handleChange}
/>

এখানে JS dynamic object key ব্যবহার হচ্ছে 🔥

function SearchBox() {

  return (
    <div>
      <input
        value={query}

    </div>
  );
}
7) ⚠️ Common Mistakes
❌ no value binding
<input onChange={...} />

এটা uncontrolled হয়ে যায়.

❌ object overwrite bug
setForm({
  name: e.target.value
});

এতে email হারিয়ে যাবে ❌

✅ fix
setForm({
  ...form,
  name: e.target.value
});
