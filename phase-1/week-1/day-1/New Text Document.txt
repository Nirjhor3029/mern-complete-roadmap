1) 🧠 Scope = Variable কোথায় accessible

Scope মানে variable কোথা থেকে access করা যাবে।

JavaScript এ mainly 3 types:

✅ Global Scope
const appName = "MERN Journey";

function show() {
  console.log(appName);
}

সব জায়গা থেকে access করা যায়।

✅ Function Scope
function user() {
  const name = "Nirjhor";
  console.log(name);
}

name শুধু function এর ভিতরে।

✅ Block Scope
if (true) {
  let age = 25;
}

age block এর বাইরে পাওয়া যাবে না।

2) ⚔️ var vs let vs const
✅ var
if (true) {
  var x = 10;
}
console.log(x); // 10

var block scope মানে না ❌
এটা function scoped.

🚨 Why dangerous

React loop / async code এ massive bugs দেয়.

✅ let
if (true) {
  let y = 20;
}
console.log(y); // error

Safer. block scoped.

✅ const
const z = 30;

reassign করা যায় না.

z = 40; // error

But object mutate করা যায়:

const user = { name: "A" };
user.name = "B";
3) 🚀 Hoisting

JavaScript execution এর আগে memory phase-এ variable declarations তুলে নেয়।

এই concept টা interview-এ খুব আসে.

✅ Example 1
console.log(a);
var a = 10;

Output:

undefined

Behind the scene:

var a;
console.log(a);
a = 10;
❗ let and const
console.log(b);
let b = 20;

এখানে error হবে.

কারণ Temporal Dead Zone (TDZ).

⚠️ Temporal Dead Zone

Declaration line এর আগে access করলে error.

{
  console.log(score); // error
  let score = 100;
}

TDZ = memory-তে আছে but access blocked.

এটা senior-level JS understanding.

💣 Interview Trap #1
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}

Output?

3
3
3

কারণ var function scoped.

✅ Fix with let
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}

Output:

0
1
2

এই concept React async bugs এ directly লাগে.

💼 Real Engineer Insight

Production code rule:

default → const
if reassign লাগে → let
avoid → var

Senior engineers প্রায় never var use করে.
