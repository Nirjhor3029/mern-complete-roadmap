1) 🧠 this Explained

this = current execution context object

Rules of this
Global context → Browser: window, Node: global
Function call → undefined in strict mode, otherwise global object
Method call → object owning method
Constructor (new) → new instance object
Arrow function → lexical this
2) Function vs Method
function sayHi() {
  console.log(this);
}

sayHi(); // global / undefined (strict mode)

const obj = { name: "Nirjhor", greet: sayHi };
obj.greet(); // obj
Function → default binding
Method → object binding
3) Constructor this
function Person(name) {
  this.name = name;
}

const p = new Person("Nirjhor");
console.log(p.name); // "Nirjhor"
new creates a new object
this points to that object
4) Arrow Function this
Arrow function does not create its own this
Lexical this → parent scope
const obj = {
  name: "Nirjhor",
  greet: () => console.log(this.name)
};
obj.greet(); // undefined (arrow uses lexical `this` = global)

Correct way:

const obj = {
  name: "Nirjhor",
  greet() {
    const inner = () => console.log(this.name);
    inner();
  }
};
obj.greet(); // "Nirjhor"
5) Bind / Call / Apply
5.1 .call(thisArg, arg1, arg2, …)
function greet(greeting) {
  console.log(`${greeting}, I am ${this.name}`);
}
const user = { name: "Nirjhor" };
greet.call(user, "Hello"); // Hello, I am Nirjhor
5.2 .apply(thisArg, [argsArray])
greet.apply(user, ["Hi"]); // Hi, I am Nirjhor
5.3 .bind(thisArg, arg1, …)
const greetUser = greet.bind(user, "Hey");
greetUser(); // Hey, I am Nirjhor
.bind() → new function permanently bound
6) Common JS / React Traps
6.1 Event Handlers
class Button {
  constructor(name) { this.name = name; }
  click() { console.log(this.name); }
}

const btn = new Button("Save");
setTimeout(btn.click, 100); // undefined
setTimeout(btn.click.bind(btn), 100); // Save
6.2 Arrow inside Method
const obj = {
  name: "Nirjhor",
  greet() {
    setTimeout(() => console.log(this.name), 100);
  }
};
obj.greet(); // Nirjhor
